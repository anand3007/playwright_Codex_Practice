Confirmed Defects

High A11Y-DEF-001

Testimonial video links do not have a discernible accessible name

Axe reported serious `link-name` failures on the testimonial media links in multiple browsers. Screen-reader users may be unable to distinguish which testimonial video is being opened.

  • Area: Hear from Our Clients
  • WCAG: 2.4.4, 4.1.2
  • Evidence: Firefox and WebKit desktop Axe runs
High A11Y-DEF-002

Desktop navigation targets are too small or too tightly spaced

Serious `target-size` issues were reported for expanded navigation items such as `AI Force`, `AI Foundry`, and `Kinetic AI`. The hit areas do not meet WCAG 2.5.8 minimums.

  • Area: Expanded main navigation
  • WCAG: 2.5.8
  • Evidence: Firefox and WebKit desktop Axe runs
High A11Y-DEF-003

Mobile footer `Global Presence` target overlaps the nearby expand control

The `Global Presence` link was flagged as partially obscured with only `1.4px` of safe clickable height in the artifact, which makes tapping unreliable on mobile.

  • Area: Footer, mobile layout
  • WCAG: 2.5.8
  • Evidence: Firefox, Chromium, and WebKit mobile Axe runs
Medium A11Y-DEF-004

Skip link does not reliably move focus into main content

The skip link is present, but the automated keyboard verification showed that focus did not reliably land in the main content area after activation.

  • Area: Keyboard bypass navigation
  • WCAG: 2.4.1
  • Evidence: Firefox and WebKit keyboard-flow failures

Test-Harness Notes

Test Note

Desktop `Menu` trigger expectation does not match the live desktop layout

The failing menu tests expected a visible `Menu` trigger, but the failure artifacts show the main navigation is already expanded on desktop and the locator resolved to hidden text.

Test Note

AI widget `Clear` button was checked before the widget entered a clearable state

The default page snapshot exposed `Submit` but not `Clear`, so this should be treated as a scenario timing issue until the test is updated to interact with the widget first.

Test Note

Chromium remains noisier than Firefox/WebKit for this site on this machine

Chromium has already shown transport instability against `hcltech.com`, so Firefox is the safer baseline browser for confirming fresh accessibility regressions.
